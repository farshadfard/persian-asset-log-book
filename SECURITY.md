# Security Policy

## Supported Versions

Security fixes are handled on the `master` branch.

## Reporting a Vulnerability

Please do not open a public issue for sensitive security reports.

If you find a vulnerability that could expose user financial data, alter portfolio calculations, or compromise deployed infrastructure, contact the maintainer privately through GitHub.

## Privacy Expectations

The app is designed to keep portfolio data on the user's device. Security-sensitive changes should preserve this local-first model and avoid sending asset records, transactions, backups, or settings to external services.
